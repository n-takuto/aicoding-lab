「Claude Code ってコード書いてくれるって言うけど、実際どのくらいできるの？」という疑問に答えるべく、**マインスイーパを丸投げしてみました**。<br>
結果：**2分16秒で動くゲームが完成しました**。

## 投げたプロンプトはこれだけ

```
src/pages/games/ フォルダにブラウザで動くマインスイーパを作って。
HTML/CSS/JavaScript のみで実装（フレームワーク不要）。
- 9×9マス、地雷10個（初級）
- 左クリックで開く、右クリックで旗を立てる
- クリア・ゲームオーバーの判定あり
- リセットボタンあり
- スマホでも動くようにする
```

これだけです。あとは Claude に任せました。

> 💡 **ぶっちゃけ、このプロンプトも Claude に書いてもらいました。**
> 「ブラウザで遊べるマインスイーパを作りたいんだけど、プロンプト作って」と一言投げたら、上の箇条書きが出てきた感じです。

## Claude が実際にやったこと

プロンプトを送ると、Claude はこんな順番で動きました。

![Claudeがコードを書いている画面](/images/minesweeper/01-writing.png)

1. **まずプロジェクト構造を確認**（既存ファイルを2つ読み込み）
2. **`src/pages/games/` フォルダがないと判断 → 直接ファイルを作成**
3. **`minesweeper.astro` を一気に341行書き上げる**（約30秒）
4. **開発サーバーを起動して動作確認**
5. **CSS のバグを自分で発見して修正**
6. **タイマーのリセットバグも自分で直す**

「作りました、確認してください」ではなく、**自分でテストして、バグを見つけて、直す**まで全部やってくれたのが驚きでした。

## 341行のコードが一瞬で

差分ビューアを開くとこんな感じ。緑色がすべて Claude が書いたコードです。

![差分ビューア（341行追加）](/images/minesweeper/05-diff.png)

ちなみに頼んでいない機能まで実装されていました。

| 実装してくれた機能 | 頼んだ？ |
|--------------|--------|
| 9×9ボード、地雷10個 | ✅ 頼んだ |
| 左クリックで開く、右クリックで旗 | ✅ 頼んだ |
| クリア・ゲームオーバー判定 | ✅ 頼んだ |
| リセットボタン | ✅ 頼んだ |
| スマホ対応 | ✅ 頼んだ |
| **タイマー（初回クリックで開始）** | ❌ 頼んでない |
| **旗カウンター（残り旗数を表示）** | ❌ 頼んでない |
| **バイブ対応（地雷を踏んだら振動）** | ❌ 頼んでない |

「頼んでないのに気を利かせて実装する」のが Claude らしいです。

## 内蔵ブラウザでそのまま確認

デスクトップ版には**内蔵ブラウザ**が搭載されていて、作ったものをアプリの中でそのまま確認できます。

![内蔵ブラウザでマインスイーパが動いている](/images/minesweeper/02-preview.png)

「コードを書く → ブラウザで確認 → 修正」という往復をアプリ内で完結できるのが便利です。

## 2分16秒の内訳

| フェーズ | 時間 |
|---------|------|
| プロジェクト確認 | 約10秒 |
| コード生成（341行） | 約30秒 |
| 開発サーバー起動 | 約20秒 |
| 動作確認・バグ修正 | 約75秒 |
| **合計** | **約2分16秒** |

人間がゼロからマインスイーパを書いたら、スマホ対応・タイマー・旗カウンター込みで**早くても2〜3時間**はかかります。それが2分16秒。

## 「丸投げ」のコツ

やってみて気づいたポイントを3つ。

**① 要件は箇条書きで渡す**

「マインスイーパ作って」だけでも作ってくれますが、欲しい機能を箇条書きで渡すと精度が上がります。

**② モードは「編集を承認」でOK**

「Auto accept edits（編集を承認）」モードにしておくと、Claude が自分でどんどん進めてくれます。プランモードだと確認ステップが増えて時間がかかります。

**③ 途中で止めない**

Claude が「動作確認します」と言い始めたら、そのまま見守りましょう。自分でバグを見つけて直してくれます。

## スマホの不具合もClaudeで直した

公開後にスマホで遊んでみたら、**旗を立てる操作がうまく動かない**不具合が出ました。

元々は「長押しで旗を立てる」仕様だったんですが、長押しの判定が不安定で、長すぎると旗が立てられないという問題が。

そこで Claude に相談しながら原因を調査したところ、タッチ中に DOM が再生成されて `touchcancel` が発火していたことが判明。最終的に「長押し」をやめて **モード切替ボタン** を追加する方向に変更しました。

![モード切替ボタンが追加されたスマホ画面](/images/minesweeper/06-mode-button.png)

ボタンをタップするたびに **💣 開くモード** ↔ **🚩 旗モード** が切り替わります。長押しより断然使いやすい。

> 💡 「作る」だけじゃなく「直す」作業も Claude に相談しながらできるのが、Claude Code の面白いところだと思います。
